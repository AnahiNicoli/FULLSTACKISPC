  • Actores: Administrador

  • Disparador: Necesidad de Consultar sobre acciones y estadísticas de las tendencias de los clientes.

  • Descripción: Caso de Uso en donde el Administrador del sistema podrá consultar las acciones más comunes de los Clientes del sistema.

  • Pre-condición: El Administrador desea hacer uso de una herramienta que le permitirá tomar decisiones con respecto al negocio.

  • Post-condición: El Administrador logra obtener la información de valor entregada por el sistema.

Flujo Normal

  1. El sistema muestra el resultado de acciones mostrando una pantalla que permite filtrar datos sobre las acciones ejecutadas por los clientes.
  2. El Administrador filtra los datos para obtener el resultado que necesita.
  3. El sistema ofrece los datos filtrados y gráficos que ilustran éstos datos.
