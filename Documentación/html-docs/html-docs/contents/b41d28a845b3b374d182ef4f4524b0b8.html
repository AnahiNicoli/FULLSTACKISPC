  • Actores: Cliente y Usuarios externos.

  • Disparador: Necesidad de seleccionar forma de pago para concretar la compra.

  • Descripción: Caso de Uso en donde se selecciona el método de pago para concretar la compra del usuario que se encuentra en interacción con el sistema.

  • Pre-condición: El usuario opta por elegir una forma de pago de los ítems seleccionados en el carrito.

  • Post-condición: Procede tras el ingreso de los datos requeridos, de lo contrario se ofrece una alerta con opción de completar datos.

Flujo Normal

  1. El sistema ofrece elegir entre pagar con débito y/o con tarjeta.
  2. El usuario selecciona un método de pago.
  3. EL sistema devuelve un formulario para completar datos de la tarjeta.

Flujo Alternativo

1. El usuario sale de la sesión sin seleccionar modo de pago.

  1. El sistema envía un mensaje de correo electrónico.

Flujo Alternativo

1. El usuario selecciona el modo de pago equivocado.

  1. El sistema permite cambiar el modo de pago.
