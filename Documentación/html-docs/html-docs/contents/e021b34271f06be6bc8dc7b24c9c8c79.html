  • Actores: Administrador

  • Disparador: Necesidad del Administrador crear una nuevo Combo.

  • Descripción: Caso de Uso en donde se accede a crear una Combo nuevo.

  • Pre-condición: El administrador debe loguearse para acceder.

  • Post-condición:

  • Si la creación del nuevo Combo es exitosa, se almacena en la base de datos y se agrega a la lista de Combos existentes en la sección de gestión de Combos.
  • Si la creación de un nuevo Combo falla, se muestra un mensaje de error que indica el problema y se le da al administrador la oportunidad de corregir los errores y volver a intentar guardar el Combo.

Flujo Normal

  1. El administrador selecciona la opción de crear una nuevo Combo.
  2. El sistema muestra un formulario para crear un nuevo Combo, que incluye campos para el nombre del Combo.
  3. El administrador ingresa los detalles del nuevo Combo.
  4. El administrador guarda el Combo nuevo.
  5. El sistema valida que los campos obligatorios estén completos.
  6. El sistema almacena al combo nuevo en la base de datos.
  7. El sistema muestra un mensaje de confirmación de que el Combo nuevo se ha creado correctamente.

Flujo Alternativo

. Si el administrador desea cancelar la creación del Combo nuevo:

  1. El administrador hace clic en el botón "Cancelar".
  2. El sistema descarta el trabajo y regresa a la lista de Combos existentes.
