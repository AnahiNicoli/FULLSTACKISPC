Disparador: Necesidad de concretar la compra.

Descripción: Caso de Uso en donde se concreta la compra del usuario que se encuentra en interacción con el sistema.

Pre-condición: El usuario desea elegir el medio de pago de los ítems seleccionados en el carrito.

Post-condición: Si el usuario ya posee cuenta, procede tras la verificación; de lo contrario se ofrece crear una cuenta con opción a utilizar su cuenta de Google.

Flujo Normal

El sistema verifica tipo de usuario (Cliente, Administrador). EL sistema verifica los datos personales. El sistema autocompleta los datos personales. El usuario opta por elegir una forma de pago.

Flujo Alternativo

1. El usuario no se encuentra logueado.

  1. El sistema devuelve un formulario en donde requiere datos a completar por el usuario, para terminar de crear la cuenta.
  2. El usuario completa datos requeridos por el sistema y confirma la carga.
  3. El sistema envía un mail de confirmación a la casilla del usuario para verificar cuenta.
  4. El usuario accede al mail y confirma creación de cuenta mediante ingreso a link proporcionado.
  5. Prosigue en punto 1 del flujo normal.

3. El usuario no ingresa datos personales obligatorios.

  1. El sistema envía un mensaje recordando ingresar los datos obligatorios.
  2. Prosigue punto 4 del flujo normal.

4. El usuario sale de la sesión sin seleccionar modo de pago.

  1. El sistema envía un mensaje de correo electrónico.
