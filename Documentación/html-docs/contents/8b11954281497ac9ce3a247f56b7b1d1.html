  • Actores: Administrador

  • Disparador: Necesidad del Administrador realizar eliminación de las publicidades.

  • Descripción: Caso de Uso en donde se accede a eliminar una Publicidad existente.

  • Pre-condición: El administrador debe loguearse para acceder.

  • Post-condición:

  • Si la eliminación de la publicidad es exitosa, se actualiza en la base de datos y en la lista de publicidades existentes en la sección de gestión de publicidad.
  • Si la eliminación de la publicidad falla, se muestra un mensaje de error que indica el problema y se le da al administrador la oportunidad de corregir los errores y volver a intentar guardar la publicidad.

Flujo Normal

  1. El administrador selecciona la opción de eliminar una publicidad existente.
  2. El sistema muestra un diálogo para confirmación de la acción la publicidad, que incluye campos para el nombre de la publicidad, la imagen, el enlace y la fecha de inicio y fin de la publicidad.
  3. El sistema busca los datos de la publicidad seleccionada.
  4. El sistema elimina la publicidad en la base de datos y actualiza el listado de publicidades.
  5. El sistema muestra un mensaje de confirmación de que la publicidad se ha actualizado correctamente.

Flujo Alternativo

. Si el administrador desea cancelar la creación de la nueva publicidad:

  1. El administrador hace clic en el botón "Cancelar".
  2. El sistema descarta eliminar la publicidad y regresa a la lista de publicidades existentes.
