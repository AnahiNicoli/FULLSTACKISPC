Disparador: Necesidad de concretar la compra mediante tarjeta.

Descripción: Caso de Uso en donde se concreta la compra del usuario que se encuentra en interacción con el sistema mediante el pago con tarjeta.

Pre-condición: El usuario selecciona abonar con tarjeta los ítems seleccionados en el carrito.

Post-condición: Si el usuario ingresa los datos de la tarjeta procede tras la verificación de lo contrario se envía una alerta.

Flujo Normal

El sistema verifica tipo de usuario (Cliente, Administrador). EL sistema solicita ingresar los datos de la tarjeta. El usuario ingresa los datos de la tarjeta. El sistema verifica los datos de la tarjeta. El sistema envía un mensaje de compra exitosa. El sistema envía un mail y confirma la concresión de la compra. El usuario puede ver los detalles de la compra en el carrito.

Flujo Alternativo

1. El usuario ingresa mal los números de la tarjeta.

  1. El sistema devuelve un alerta que pide revisar y editar datos para terminar de concretar el pago.
  2. Prosigue en punto 3 del flujo normal.

3. El usuario no ingresa los números de la tarjeta.

  1. El sistema envía un mensaje recordando ingresar los datos obligatorios.
  2. Prosigue punto 3 del flujo normal.

4. El usuario sale de la sesión sin confirmar la compra.

  1. El sistema envía un mensaje de correo electrónico.
