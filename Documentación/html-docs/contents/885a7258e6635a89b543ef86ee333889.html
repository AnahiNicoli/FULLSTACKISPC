  • Actores: Administrador

  • Disparador: Necesidad del Administrador realizar eliminación de Combos.

  • Descripción: Caso de Uso en donde se accede a eliminar un Combo existente.

  • Pre-condición: El administrador debe loguearse para acceder.

  • Post-condición:

  • Si la eliminación del Combo es exitosa, se actualiza en la base de datos y en la lista de Combos existentes en la sección de gestión de Combos.
  • Si la eliminación del Combo falla, se muestra un mensaje de error que indica el problema y se le da al administrador la oportunidad de corregir los errores y volver a intentar guardar el Combo.

Flujo Normal

  1. El administrador selecciona la opción de eliminar un Combo existente.
  2. El sistema muestra un diálogo para confirmación de la acción.
  3. El sistema busca los datos el Combo seleccionado.
  4. El sistema elimina el Combo en la base de datos y actualiza el listado de Combos.
  5. El sistema muestra un mensaje de confirmación de que el Combo se ha actualizado correctamente.

Flujo Alternativo

. Si el administrador desea cancelar la eliminación del Combo:

  1. El administrador hace clic en el botón "Cancelar".
  2. El sistema descarta eliminar el Combo y regresa a la lista de Combos existentes.
