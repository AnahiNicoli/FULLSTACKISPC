  • Actores: Administrador

  • Disparador: Necesidad del Administrador realizar gestiones sobre las publicidades ofrecidas por la empresa.

  • Descripción: Caso de Uso en donde se accede a la gestión de publicicdades(Crear,Modificar,Eliminar).

  • Pre-condición: El administrador debe loguearse para acceder.

  • Post-condición:

  • Si la creación de la nueva publicidad es exitosa, se almacena en la base de datos y se agrega a la lista de publicidades existentes en la sección de gestión de publicidad.
  • Si la creación de la nueva publicidad falla, se muestra un mensaje de error que indica el problema y se le da al administrador la oportunidad de corregir los errores y volver a intentar guardar la publicidad.

Flujo Normal

  1. El administrador selecciona la opción de crear una nueva publicidad.
  2. El sistema muestra un formulario para crear una nueva publicidad, que incluye campos para el nombre de la publicidad, la imagen, el enlace y la fecha de inicio y fin de la publicidad.
  3. El administrador ingresa los detalles de la nueva publicidad.
  4. El administrador guarda la nueva publicidad.
  5. El sistema valida que los campos obligatorios estén completos y que la fecha de inicio no sea posterior a la fecha de fin.
  6. El sistema almacena la nueva publicidad en la base de datos.
  7. El sistema muestra un mensaje de confirmación de que la nueva publicidad se ha creado correctamente.

Flujo Alternativo

. Si el administrador desea cancelar la creación de la nueva publicidad:

  1. El administrador hace clic en el botón "Cancelar".
  2. El sistema descarta la nueva publicidad y regresa a la lista de publicidades existentes.
