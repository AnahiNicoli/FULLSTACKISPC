  • Actores: Administrador

  • Disparador: Necesidad del Administrador realizar modificaciones sobre las publicidades.

  • Descripción: Caso de Uso en donde se accede a modificar una publicidad existente.

  • Pre-condición: El administrador debe loguearse para acceder.

  • Post-condición:

  • Si la modificación de la publicidad es exitosa, se actualiza en la base de datos y en la lista de publicidades existentes en la sección de gestión de publicidad.
  • Si la modificación de la publicidad falla, se muestra un mensaje de error que indica el problema y se le da al administrador la oportunidad de corregir los errores y volver a intentar guardar la publicidad.

Flujo Normal

  1. El administrador selecciona la opción de modificar una publicidad existente.
  2. El sistema busca los datos de la publicidad seleccionada.
  3. El sistema muestra un formulario para modificar la publicidad, que incluye campos para el nombre de la publicidad, la imagen, el enlace y la fecha de inicio y fin de la publicidad.
  4. El administrador realiza las modificaciones pertinentes en la publicidad.
  5. El administrador resgualda los cambios realizados en la publicidad.
  6. El sistema valida que los campos obligatorios estén completos y que la fecha de inicio no sea posterior a la fecha de fin.
  7. El sistema actualiza la publicidad en la base de datos.
  8. El sistema muestra un mensaje de confirmación de que la publicidad se ha actualizado correctamente.

Flujo Alternativo

. Si el administrador desea cancelar la creación de la nueva publicidad:

  1. El administrador hace clic en el botón "Cancelar".
  2. El sistema descarta la modificación de publicidad y regresa a la lista de publicidades existentes.
